== Guides

You can access to 2 different parts during this lab:

* A first introduction to OpenShift Virtualization (Beginner's Guide) focus on how a virtualization user can easily deploy VM via the OpenShift Console.
* A more advanced guide (Advanced guide) focus on VM optimization and placement

*Feel free to follow the one you wanted.*

== Cluster Access

Each of you will have a dedicated OpenShift cluster during this lab.
You will have 2 accouts: 

* A simple user account with limited view on OpenShift infrastructure
* A admin account with full vien on your OpenShift cluster

This strict user access control highlights OpenShift's powerful multi-tenancy.
Many projects can be operated on a single OpenShift cluster without interfering with each other.

You will complete this entire workshop via the OpenShift console.


[#labenvironment]
== Lab Environment

The lab environment is running Red Hat OpenShift 4.20 with the following required operators pre-installed:

* OpenShift Virtualization
* OpenShift Data Foundations
* Node Health Check Operator
* Self Node Remediation Operator
* Kube Descheduler Operator
* Migration Toolkit for Containers
* Migration Toolkit for Virtualization

include::lab-access.adoc[]

[IMPORTANT]
====
The environments for these labs are generated on-demand and are not representative of reality. Indeed, for cost and automation reasons, these OpenShift Virtualization environments are "nested" and not supported in production. Performance may therefore be significantly affected.
====
