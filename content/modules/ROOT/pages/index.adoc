== Cluster Access

Your workshop environment is located on a shared cluster.
You will be an OpenShift user and not a cluster administrator. Your view will therefore be limited to your project/namespace, and the entire infrastructure part (such as the nodes where your VMs are running) will be hidden from you.
Other workshop participants are also on this cluster.
When important OpenShift features are not visible to users, this guide will provide explanations and screenshots of the relevant sections.
This strict user access control highlights OpenShift's powerful multi-tenancy.
Many projects can be operated on a single OpenShift cluster without interfering with each other.

You will complete this entire workshop via the OpenShift console.

== Lab Environment

The terminal window to your right is *already* logged into the lab environment via the bastion host as the *{ssh_user}*.
All steps of this lab are to be completed from the bastion host as the *{ssh_user}* logged in to the OpenShift cluster as *admin*, unless the OpenShift console is indicated.

The lab environment is running Red Hat OpenShift 4.20 with the following required operators pre-installed:

* OpenShift Virtualization
* OpenShift Data Foundations
* Node Health Check Operator
* Self Node Remediation Operator
* Kube Descheduler Operator
* Migration Toolkit for Containers
* Migration Toolkit for Virtualization
